### 3.5 Schema Mismatch: Standalone vs. Orchestrator

This is the primary technical divergence between the standalone script and the orchestrator version.

#### Standalone script (`Bulk/R/9_Oahu_Opportunity_Cost_Map.R`)
- Reads only R MICE datasets (`R_Imputed_Dataset_1..100.csv`)  
- The R imputed dataset has column `final_acreage`
- Computes OC: `opp_cost = final_acreage * Baseline_Value_Per_Acre` ✅

#### Orchestrator version (`run_9_` in Phase_6.R)
- Reads **all three** language groups (R + Py + Jl) via `pool_oahu_oc()`
- The Py imputed dataset has columns: `osm_acreage, acreage_source` (not `final_acreage`)
- The orchestrator's `pool_oahu_oc()` function uses: `opp_cost = final_acreage * Baseline_Value_Per_Acre`

> [!CAUTION]
> **Column name mismatch for Python imputed datasets.** The `pool_oahu_oc()` helper inside `run_9_` references `final_acreage` for all three language groups. However, the Python imputed datasets use the column name `osm_acreage`, not `final_acreage`. When `pool_oahu_oc()` is called with `PY_IMPUTED_PATHS`, every `mutate(opp_cost = final_acreage * Baseline_Value_Per_Acre)` will produce **all-`NA` results** because `final_acreage` does not exist in those files.
>
> The R imputed datasets use `final_acreage`. The Julia imputed datasets — which use the same schema as Python (`osm_acreage`) — will also silently produce all-NA values.
>
> The result: `pooled_py` and `pooled_jl` will be entirely NA, and the Grand Mean will degrade to just the R pool. The map will still render but will be based on R-only data. This is a silent data quality failure, not a hard stop.

## Part 4 — Script 15: `15_Residual_Map.R` (Standalone) vs. `run_15_Residual_Map()` (Orchestrator)

### 4.1 What It Produces

Script 15 maps **OLS regression residuals** (actual acreage vs. model-predicted acreage) at the county level:
- **Map 15.1:** `log(actual_acreage) − log(predicted_acreage)` — mean log-residual per county (statistical spatial diagnostic)
- **Map 15.2:** `(actual_acreage − exp(predicted_acreage)) × Baseline_Value_Per_Acre` — sum dollar-residual per county (economic interpretation)

The prediction model is: `log(final_acreage) = β₀ + β₁·Holes + β₂·I(Urban)`

## Part 5 — Error Handling / Silent Failure Analysis

### 5.1 Standalone Scripts

**Neither standalone script has any `tryCatch`, `try()`, `silent = TRUE`, or `suppressWarnings()` wrappers around their execution logic.** Both use an explicit input-guard pattern:

```r
for (f in c(...)) {
    if (!file.exists(f)) stop(sprintf("[FATAL] Input not found:\n  %s", f))
}
```

Since all input files exist, these guards will pass silently. There are no mechanisms that would cause either standalone script to run without producing output unless (a) a column was missing and caused an error partway through, or (b) `ggsave()` failed due to a write permission issue. Neither condition appears to apply.

### 5.2 Orchestrator Versions

The orchestrator's `run_9_` and `run_15_` functions follow the same input-guard pattern. They also do not wrap their internals in `tryCatch`. However:

> [!WARNING]
> The `pool_oahu_oc()` helper in `run_9_` will **not throw an error** when Python/Julia imputed datasets return all-NA for `final_acreage` — it will silently produce NA-valued pooled estimates for those language groups. The `full_join` → `rowMeans(..., na.rm = TRUE)` pipeline will degrade gracefully to R-only, but **no warning or diagnostic message will be emitted** about the NA collapse. This is the only source of silent failure risk in the current code.

```r
mutate(opp_cost = final_acreage * Baseline_Value_Per_Acre)
```

This needs to be made column-agnostic, e.g.:

```r
mutate(
    acreage   = coalesce(final_acreage, osm_acreage),  # R uses final_acreage; Py/Jl use osm_acreage

)
```

Or pass the acreage column name as a parameter to `pool_oahu_oc()`.
